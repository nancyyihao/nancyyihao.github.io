为什么这样是可行的呢？其实是Java 5之后对volatile关键字做了修改，它能够禁止编译器进行指令重排，简单地说就是加了volatile关键字之后，编译器能够保证1–>2–>3的执行顺序，因此也就没有上述问题了。更多关于volatile关键字，点击：http://www.cnblogs.com/dolphin0520/p/3920373.html
